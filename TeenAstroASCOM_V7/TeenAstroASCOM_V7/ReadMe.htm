TeenAstro ASCOM Driver

Welcome to the TeenAstro ASCOM Driver! This driver allows astronomical software to communicate with the TeenAstro Telescope Mount Controller.

Important Compatibility Notice: This version of the ASCOM driver requires TeenAstro Firmware 1.6.x or newer. It relies on the new high-speed :GXAS# and :GXCS# binary protocol commands which are not present in older firmware versions (1.5.x or below). If you are using older firmware, you must upgrade your TeenAstro Main Unit, Focuser, and Hand Controller firmware, or continue using the legacy ASCOM driver.

What This Driver Can Do

The TeenAstro ASCOM Driver acts as a "Local Server" (hub). This means it can support multiple ASCOM clients (like planetariums, guiding software, and sequence generators) connecting to your TeenAstro hardware simultaneously without conflicting with each other.

The driver provides two ASCOM device interfaces via a single shared connection:

  • Telescope Driver: Controls the mount's tracking, slewing, parking, pulse guiding, and synchronization. It correctly translates ASCOM standard commands into the enhanced LX200 protocol used by TeenAstro.
  • Focuser Driver: Controls the integrated focuser supported by the TeenAstro main unit or Smart Hand Controller. It supports absolute positioning, halt commands, and temperature reporting (if a sensor is equipped).

Key Features

  • Shared Connection: Both the Telescope and Focuser use the exact same COM Port or IP Address. You configure it once, and both drivers share the underlying communication channel seamlessly.
  • Serial & IP Support: Connect via a standard USB/Serial COM port, or over the network using Wi-Fi / Ethernet via IP.
  • High-Performance State Polling: Uses the TeenAstro-specific :GXAS# bulk status command to efficiently poll all mount states, tracking rates, and coordinates in a single fast transaction.
  • Diagnostic Logging: Built-in trace logging to help troubleshoot communication or protocol issues. Logs are saved in your Documents\ASCOM\Logs yyyy-mm-dd folder.

What's New in Version 1.6 (vs 1.5)

This release marks a major architectural overhaul of the TeenAstro software ecosystem, delivering significantly higher performance and stability for ASCOM users. Key improvements include:

  • Pure C# Local Server Architecture: The legacy Native C++ DLL (`TeenAstroAscomNative`) and Visual Basic drivers have been completely removed and replaced by a modern, unified C# Local Server implementation. This resolves memory management issues, improves cross-process stability when multiple astronomy applications (e.g., N.I.N.A., PHD2, Stellarium) connect at once, and natively supports 64-bit applications without relying on translation layers.
  • New Integrated Focuser Driver: Added a dedicated ASCOM Focuser driver (supporting IFocuserV3) that shares the exact same connection (COM or IP) as the Telescope driver. It fully supports absolute positioning, halting, and accurate temperature reading. Crucially, the driver now strictly adheres to ASCOM Conformance standards by gracefully handling missing temperature sensors (using the firmware's -99.99 sentinel) without throwing unexpected COM exceptions that crash clients. Focuser initialization timing was also fixed at the firmware level for reliable Dallas 1-Wire sensor discovery.
  • High-Speed Bulk Polling (GXAS & GXCS): The driver no longer relies on dozens of slow, individual LX200 property queries (e.g., `:GR#`, `:GD#`, `:GXI#`). Instead, it uses the new TeenAstro-exclusive :GXAS# (status) and :GXCS# (config) commands. These commands return a single Base64-encoded, checksum-validated binary packet containing all mount coordinates, tracking rates, focuser state, guiding state, and system time in one lightning-fast transaction. This vastly reduces serial/TCP latency and ensures that coordinates and state properties are perfectly synchronized.
  • ASCOM Conformance & Float64 Precision: Tracking rates (RA, Dec, HA) and guide rates are now handled using native 64-bit floating point math (`double`) across the entire stackâ€”from the Main Unit firmware up through the ASCOM driver. This eliminates rounding errors in custom tracking rates and resolves multiple ASCOM Conform testing issues related to round-trip rate accuracy, StepSize, and tracking state reporting during slews.
  • Improved Guiding, Slewing & Tracking (Firmware):
    • Resolved a long-standing RA East/West pulse guiding asymmetry that occurred when tracking was active.
    • Fixed goto RA/Dec drift calculation issues caused by main loop stalls (e.g., when delayed by focuser communications).
    • Fixed an issue where UTC Date read-modify-restore operations could interrupt Declination slews.
    • Removed deprecated/orphan get commands that slowed down the command parser.
  • Smart Hand Controller (SHC) & Emulator: Extensive fixes to the SHC UI responsiveness during slews, web server crash fixes, and the introduction of a new Windows-based TeenAstro Emulator & Flutter App for offline testing and configuration.

How to Use

  1. Install the driver using the ASCOM Installer.
  2. Open your ASCOM-compatible astronomy software (e.g., N.I.N.A., PHD2, Stellarium, Cartes du Ciel).
  3. In the software's equipment configuration, choose TeenAstro Telescope for the mount and TeenAstro Focuser for the focuser.
  4. Click the Properties / Setup button for either device to open the settings window.
  5. Select your connection type (Serial or IP). If Serial, choose the correct COM Port. If IP, enter the IP address of your TeenAstro controller and Port (usually 9999).
  6. Click OK, and connect your software to the devices.

Troubleshooting

If you experience connection issues or unexpected behavior:

  • Open the Setup dialog for the TeenAstro Telescope and check Enable Trace Logging.
  • Try to connect again and reproduce the issue.
  • Check the logs in your Documents\ASCOM\Logs directory for detailed communication traces between the driver and the TeenAstro hardware.
