Mount Alignment — CoordConv

TeenAstroCoordConv implements a two-star alignment model based on Toshimi Taki’s method (Sky & Telescope, Feb 1989, pp. 194–196). The raw transformation is refined using Singular Value Decomposition to project onto the nearest proper rotation matrix (det = +1), compensating for measurement noise.

Source: libraries/TeenAstroCoordConv/

CoordConv Class

class CoordConv : public LA3 {
public:
    // Reference star axis angles
    double ax1[2], ax2[2];

    // Transformation matrices
    double T[3][3];     // HO → IN (sky → instrument)
    double Tinv[3][3];  // IN → HO (instrument → sky)

    // SVD factors
    double u[3][3], v[3][3];

    CoordConv();
    void reset();
    void clean();
    bool isReady() const;
    double getError() const;
    unsigned char getRefs() const;

    // Persistence
    void getT(float &m11, ..., float &m33);  // read T as floats
    void setT(float m11, ..., float m33);       // restore T from EEPROM
    void setTinvFromT();                        // recompute Tinv = T¹

    // Alignment procedure
    void addReference(double angle1, double angle2, double axis1, double axis2);
    bool calculateThirdReference();

    // Post-alignment refinement
    void minimizeAxis1(double offset);
    void minimizeAxis2();

protected:
    double dcAARef[3][3];   // axis direction cosines (3 refs × 3 components)
    double dcHDRef[3][3];   // sky direction cosines
    unsigned char refs;     // reference count (0, 1, or 2)
    bool isready;
    double anglediff;       // angular consistency error

    void buildTransformations();
};

Taki Alignment Method

Step 1 — Collect Two Reference Stars

For each reference star i ∈ {0, 1}, record:

  • Sky position: (angle1, angle2) = (Azimuth, Altitude)
  • Instrument position: (axis1, axis2) = physical mount axes

Convert each pair to direction cosines on the unit sphere:

d⃗_HD,i = toDirCos(Az_i, Alt_i) ← sky d⃗_AA,i = toDirCos(axis1_i, axis2_i) ← instrument

Step 2 — Compute Third Reference (Cross Product)

A third, synthetic reference point is computed as the normalized cross product of the first two:

d⃗_HD,2 = normalize( d⃗_HD,0 × d⃗_HD,1 ) d⃗_AA,2 = normalize( d⃗_AA,0 × d⃗_AA,1 )

This completes a full 3D basis for both the sky and instrument frames.

Step 3 — Build Transformation Matrix

Form 3×3 matrices from the direction cosines (each row is one reference):

D_HD = [ d⃗_HD,0 ] D_AA = [ d⃗_AA,0 ] [ d⃗_HD,1 ] [ d⃗_AA,1 ] [ d⃗_HD,2 ] [ d⃗_AA,2 ]

The transformation from sky to instrument is:

T = D_AAᵀ · (D_HDᵀ)⁻¹

And the inverse:

T_inv = T⁻¹ = D_HDᵀ · (D_AAᵀ)⁻¹

Angular Error

The consistency of the reference stars is checked by comparing the angular separation in both frames:

anglediff = angle(d⃗_HD,0, d⃗_HD,1) − angle(d⃗_AA,0, d⃗_AA,1)

A large anglediff indicates misidentified stars or pointing errors.

SVD Correction — Optimal Rotation

Problem

Due to measurement noise and star position errors, the matrix T from the Taki method may not be a proper rotation (its determinant may not be exactly +1). We need to find the nearest proper rotation matrix.

Solution

Compute the SVD of Tinv:

T_inv = U · Σ · Vᵀ

Then replace Σ with a diagonal matrix D that forces det = +1:

D = diag(1, 1, det(U) · det(V)) R_opt = U · D · Vᵀ
  • If det(U) · det(V) = +1: D = I, so Ropt = U · VT (already a rotation).
  • If det(U) · det(V) = −1: D = diag(1, 1, −1), flipping the sign of the smallest singular value’s contribution to ensure det(Ropt) = +1.

Implementation

getsvd(Tinv, u, v);
double du = determinant(u);
double dv = determinant(v);
diag[2][2] = du * dv;       // +1 or -1
multiply(tmp, diag, vt);     // tmp = D · V^T
multiply(Tinv, u, tmp);      // Tinv = U · D · V^T  (optimal rotation)
transpose(T, Tinv);          // T = Tinv^T

After this step, T and Tinv are guaranteed to be proper rotation matrices (orthogonal, det = +1), which is essential for stable coordinate conversions.

Axis Minimization

minimizeAxis1(offset)

Compensates for a systematic offset in the first axis. It extracts Euler angles from Tinv, shifts axis1 for both reference stars by the offset, recomputes direction cosines, and recalculates the third reference and transformation:

axis1'_i = direct_axis1_i − offset (for i = 0, 1) Recompute d⃗_AA,i = toDirCos(axis1'_i, axis2_i) Recompute third reference and T via calculateThirdReference()

minimizeAxis2()

Iterative Newton-like refinement (5 iterations) to minimize the angular error by adjusting axis2 for both reference stars:

For each iteration: δ = anglediff / denom where denom = −Σ_i ( dcAA_i,0 · sin(ax2_{1−i}) · cos(−ax1_{1−i}) + … ) ax2[0] += 0.8 · δ ax2[1] += 0.8 · δ Recompute direction cosines and third reference.

The 0.8 damping factor prevents oscillation.

Firmware Integration

The alignment matrices are used throughout the MainUnit real-time loop:

OperationMatrix UsedConversion
Read current sky positionT (via Tinv)IN → HO → EQ
Goto targetTinvEQ → HO → IN (target steps)
Sync mount to known positionTinvEQ → IN (set stepper positions)
Compute tracking ratesTinvEQ ±δt → IN (rate = Δaxis / Δt)
Real-time altitude for safetyTIN → HO (check altitude limits)

EEPROM Persistence

The 9 elements of T are stored as float (4 bytes each, 36 bytes total) plus a validity flag (EE_Tvalid). On power-up, if valid, T is loaded and Tinv is recomputed via setTinvFromT().
