SHC Hand Controller

The Smart Hand Controller (SHC) runs on ESP8266 and provides a local user interface with OLED display, button navigation, catalogs, and menu-driven control of the mount. It communicates with the MainUnit over serial using the LX200 protocol.

Source: TeenAstroSHC/

SmartHandController Class

Setup and Update

The main controller class orchestrates initialization and the event loop:

MethodPurpose
setup()Initialize serial, OLED (U8g2), buttons, LX200 client, load catalogs
update()Main loop: read buttons, process menu, send LX200 commands, refresh display

Key Methods

  • processKey() — Handle button presses (up/down/select/back)
  • updateDisplay() — Refresh OLED content for current page
  • updateMount() — Poll MainUnit for state (GXAS/GXCS)
  • setLX200() — Send LX200 command to MainUnit

Menu System Architecture

U8g2_ext

The display uses U8g2 (and extensions in U8g2ext) for OLED rendering. Text, menus, and selection lists are drawn via U8g2 primitives.

Selection Lists

Menus use selection lists: a scrollable list of items with a highlighted selection. The user navigates with up/down and selects with the center button.

Top-Level Menus → Sub-Menus

The menu hierarchy is structured as:

Top-level menus (Main, Catalog, Sync/Goto, Align, Settings, ...) │ ├── Sub-menus (e.g. Catalog → Messier, NGC, Stars) │ │ │ └── Actions (select object → Goto, Sync, Align) │ └── Settings sub-menus (Mount, Motor, Limits, Time, WiFi, ...)

Each menu level has its own handler; navigation state is tracked in the controller.

Actions Files

Action handlers implement the behavior for catalog browsing, goto, sync, and alignment:

FilePurpose
Actions_SyncGoto.cppSync and goto from catalog selection; menuCatalog, menuCatalogAlign
Actions_Tel.cppTelescope/catalog display; object selection and title display
Actions_Focuser.cppFocuser in/out, speed, temperature; forwards focus commands to MainUnit

Display Code

Display logic is split across several files:

  • SmartController_Display.cpp — Main display update; draws current page (RA/Dec, Alt/Az, time, etc.)
  • SmartController_Message.cpp — Message and status text overlays
  • U8g2ext — Extended U8g2 helpers for catalog scrolling, moving text, selection lists

Each page type has a dedicated draw routine that formats coordinates, time, or menu content.

LX200Client Integration

The SHC uses an LX200Client (or equivalent) to communicate with the MainUnit over serial. Commands are sent in :CMD# format; replies are parsed for display and state.

The SHC acts as an LX200 client: it sends commands and receives replies. It does not implement the full LX200 server; the MainUnit is the protocol endpoint.
