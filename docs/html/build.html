Build System

TeenAstro uses PlatformIO for firmware, Flutter for the app, and .NET/MSBuild for ASCOM, Uploader, and MSI. All build scripts are in the repo; a full clone contains everything needed to build without extra generation steps.

PlatformIO Install

pip install platformio

Flutter Install

Download from flutter.dev; add to PATH. Run flutter doctor.

.NET Install

Visual Studio 2022 with “.NET desktop development” workload, or Build Tools for Visual Studio with .NET Framework 4.7.2 targeting pack.

build_firmware.py Flags

  • --target main — MainUnit only
  • --target focuser — Focuser only
  • --target shc — SHC only (ENGLISH, FRENCH, GERMAN)
  • --list — List variants without building
  • --clean — Clean before build

build_app.ps1 Flags

  • -SkipAndroid — Windows only
  • -SkipWindows — Android only
  • -FixFlutter — Reinstall Flutter if broken
  • -FlutterPath "path" — Override Flutter location

Unit Tests

91 tests across 3 suites:

  • test_mount_status — GXAS/GXCS parsing, MountState
  • test_align_state — Alignment state machine
  • test_io — LX200 I/O, transport
pio test -d tests

Requires MinGW on PATH for native tests on Windows.

Versioning

TeenAstro uses X.Y versioning only; no patch component in the release dropdown.

Version is set in RELEASE_VERSION in build_firmware.py (e.g. 1.6).

EEPROM Auto-Init Key

EE_autoInitKey is a magic value stored in EEPROM. On first boot (or after factory reset), the key is absent or invalid; the firmware initializes default config. On subsequent boots, the key is valid and saved config is restored.

Factory reset (:$$#) erases the entire EEPROM, including the auto-init key.
