LX200 Protocol

TeenAstro uses the LX200 serial protocol (Meade Autostar/LX200 compatible) for all communication between clients (SHC, WiFi Server, Flutter app, ASCOM) and the MainUnit. Commands are case-sensitive and use a fixed format.

Command Format

All commands use the format :CMD#:

  • Leading colon :
  • Command string (case-sensitive)
  • Hash # terminator

Replies may be ACK-only (\x06), short (1/0), or variable-length data. Reply lengths are defined in CommandReplyLength.h and lx200_reply_lengths.dart.

LX200Client Class

LX200Client provides an object-oriented LX200 client for C++ firmware (SHC, WiFi bridge):

IX200Transport Interface

Abstract transport for send/receive. Implementations:

  • StreamTransport — Wraps a Stream (Serial, Serial1, etc.)

Key Methods

MethodPurpose
sendReceive(cmd, replyType, buf, size, timeout)Send command, read reply into buffer
sendReceiveAuto(cmd, cmdreply, buf, size, ...)Auto-detect reply type from CommandReplyLength
get(cmd, out, len)Send get command, return value in out
set(cmd)Send set command, return success/fail

Convenience getters: getRaStr, getDecStr, getAltStr, getAzStr, getUTCTimeStr, getSiderealStr, etc.

LX200Navigation Free Functions

LX200Navigation provides higher-level helpers built on LX200Client:

  • Equinox precession (J2000 ↔ equinox-of-date)
  • Catalog object lookup and coordinate conversion
  • Goto/sync helpers with coordinate transforms
