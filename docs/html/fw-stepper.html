Motor & Stepper Drivers

The TeenAstro stepper stack provides a unified interface for different motor driver types: classic step/dir, TMC26x, TMC2130, TMC5160, and TMC2660. The MainUnit uses these drivers to generate step pulses for each axis.

Source: libraries/TeenAstroStepper/

Class Hierarchy

MotionControl (abstract) │ ├── StepDir (step/dir pins, ISR-driven) │ └── Mc5160 (SPI motion controller, TMC5160) MotorDriver (interface) │ └── Driver (concrete implementations)

MotionControl defines the interface for moving axes; StepDir and Mc5160 implement it for different hardware.

Driver Class and MOTORDRIVER Enum

The Driver class wraps the physical driver. The MOTORDRIVER enum selects the driver type:

ValueDescription
NODRIVERNo driver (disabled axis)
STEPDIRClassic step/dir (A4988, DRV8825, etc.)
TMC26XTMC26x stepper driver
TMC2130TMC2130 SPI driver
TMC5160TMC5160 SPI motion controller
TMC2660TMC2660 driver

StepDir Class

StepDir drives steppers via step and direction pins. The timer ISR generates step pulses at the computed interval.

Key Members and Methods

  • ISR — Timer interrupt; fires at interval_Step_Cur, toggles step pin
  • programSpeed() — Sets step interval from desired speed
  • step / dir pins — GPIO for step pulse and direction

Mc5160 Class

Mc5160 uses the TMC5160 SPI motion controller. The chip handles acceleration, deceleration, and step generation internally; the firmware programs target positions and reads status over SPI.

Advantages: lower CPU load, built-in stall detection, microstep interpolation.

PositionState Enum

Motion state for each axis during goto or tracking:

StateDescription
PS_IDLENot moving
PS_ACCELAccelerating toward target
PS_CRUISEConstant speed (max slew)
PS_DECELDecelerating
PS_DECEL_TARGETFinal deceleration to target
PS_STOPPINGStopping (halt or fault)

TMC Driver Integration

TMC drivers support:

  • Current — High/low current (mA) for run and idle
  • Microstep — 1, 2, 4, 8, 16, 32, 64, 128, 256
  • Silent mode — SpreadCycle vs StealthChop (TMC2130/5160)

These are configured via LX200 set commands and stored in EEPROM.

Step Rate Math

Step interval (timer ticks between steps) and speed (steps/s) are related by:

interval2speed(interval) = ClockSpeed / interval speed2interval(V) = ClockSpeed / V

Where ClockSpeed is the timer clock in Hz (e.g. 1 MHz on Teensy).

Minimum Step Interval

PlatformStepsMinInterval
Teensy 4.x3 µs
Teensy 3.x / older12 µs

This limits maximum slew speed to avoid overloading the ISR.
