Stereographic Zenithal Projection (_Proj class)

The planetarium uses a stereographic zenithal projection to map the celestial sphere onto a flat canvas. This projection preserves angles (conformal) and maps circles on the sphere to circles (or lines) on the plane. The zenith is at the center; the horizon forms the outer boundary circle.

project(raH, decDeg) Method

Projects equatorial coordinates (RA in hours, Dec in degrees) to canvas (x, y) pixels.

Step 1: Hour Angle

HA = (LST − RA) × 15 × π/180 where HA is in radians, LST and RA in hours; 15°/h converts to degrees.

Step 2: Altitude (spherical trig)

sin(Alt) = sin(Dec) · sin(φ) + cos(Dec) · cos(φ) · cos(HA) where φ = observer latitude (radians), Dec in radians.

Step 3: Azimuth (cosine formula)

cos(Az) = ( sin(Dec) − sin(Alt) · sin(φ) ) / ( cos(Alt) · cos(φ) )

Step 4: Azimuth quadrant correction

If sin(HA) > 0: Az = 2π − Az

Ensures Az is measured correctly from North through East (0° = N, 90° = E).

Step 5: Stereographic radius

r = radius · cos(Alt) / (1 + sin(Alt)) Stereographic formula: point on sphere at altitude Alt projects to distance r from zenith.

Step 6: Canvas coordinates

x = cx − r · sin(Az + rotation) y = cy − r · cos(Az + rotation) + panY

Rotation is applied to azimuth; panY shifts the view vertically. Y-axis is flipped (minus cos) for screen coordinates.

projectAltAz(alt, az) Variant

Direct projection from horizontal coordinates (altitude, azimuth in radians). Skips the equatorial-to-horizontal conversion; useful for objects already in Alt/Az (e.g., mount position, horizon grid):

r = radius · cos(alt) / (1 + sin(alt)) x = cx − r · sin(az + rotation) y = cy − r · cos(az + rotation) + panY
Use project for catalog objects (RA/Dec); use projectAltAz for horizon grid, cardinal labels, and mount crosshair when already in Alt/Az.

CustomPainter: _SkyRenderer

_SkyRenderer extends Flutter’s CustomPainter and performs all sky-dome drawing in a single paint() call. It receives data from the app state and renders the 14 layers in back-to-front order.

14 Rendering Layers

Layers are drawn in order from back (1) to front (14). Each layer may be toggled via settings.

#LayerDetails
1 Sky background Gradient fill from zenith (darker) to horizon (lighter). In night mode: deep blue/black; day mode: lighter blue. Uses Canvas.drawRect or radial gradient with sky circle mask.
2 Ground Path.combine(PathOperation.difference, skyCircle, groundPath). The sky circle is the dome; ground is everything below the horizon. Typically a dark fill (brown/gray) or gradient simulating terrain.
3 Horizon circle Outer boundary of the visible sky. Drawn as a stroked circle at radius from center. May include a subtle glow or thicker stroke for visibility.
4 Milky Way (_drawMilkyWay) Polylines with center path + width. Data in J2000 equatorial; each vertex converted via equatorialEquinoxToJNow then projected. Clipped to sky circle (points below horizon discarded). Drawn with semi-transparent fill or stroke.
5 Alt-Az grid Altitude circles at 10°, 20°, …, 80°; azimuth lines every 30° (0, 30, 60, …, 330). Uses projectAltAz(alt, az). Circles become eccentric on stereographic projection; lines radiate from zenith.
6 Equatorial grid RA lines at 2h steps (0h, 2h, 4h, …); Dec circles at ±75°, 15° steps (−75, −60, …, +75). Each point converted J2000→JNow, then EQ→Alt/Az, then projected. Grid follows celestial coordinates.
7 Constellation lines Line segments between star pairs. Each segment endpoint: star RA/Dec in J2000 → JNow via equatorialEquinoxToJNow → project. Stroked with thin line (1–2 px).
8 Constellation names TextPainter at constellation centroid (precomputed or mean of member stars). Small font, dim color. Only drawn if above horizon and within magnitude/zoom limits.
9 Stars Magnitude limit from settings; B-V color index → color (blue for hot, red for cool). Bright stars use radial gradient (core + halo). Size clamped 0.15–22 px based on magnitude. Each star: J2000→JNow → project.
10 DSO symbols Stellarium-like symbols per type: galaxy (ellipse), planetary nebula (ring), open cluster (dots), globular cluster (filled circle with halo), nebula (irregular shape), etc. Position from DSO RA/Dec; size from catalog or fixed scale.
11 Planets Filled circles with optional halo for bright planets (Venus, Jupiter). Position from ephemeris (RA/Dec JNow). Size reflects magnitude or fixed scale.
12 Crosshair Mount RA/Dec position. Uses mount coordinates (already JNow); project to canvas. Cross or circle at center of FOV. Indicates where the telescope is pointing.
13 Target marker Diamond shape at goto target RA/Dec. Only visible when a goto is active. Project target coordinates; draw diamond with distinct color (e.g., green or cyan).
14 Cardinal labels N, NE, E, SE, S, SW, W, NW at horizon. Azimuths 0°, 45°, 90°, …. Use projectAltAz(0, az) (alt=0 at horizon). TextPainter at each point.
Layers 4, 7, 8, 9 require J2000→JNow conversion for precession/nutation. Layers 5, 14 use Alt/Az directly. Layers 11, 12, 13 use mount/target positions (already JNow).

Hit Testing

_handleTap

On tap, the engine iterates over planets, stars, and DSOs. For each object:

  1. Project its position to canvas (x, y).
  2. Compute Euclidean distance from tap point: d = √((x − tapX)² + (y − tapY)²).
  3. Pick the closest object within 30 px threshold.

Priority order: planets first (most prominent), then stars, then DSOs. The selected object is stored in selectedObject and may trigger an info panel or goto.

Gesture Handling

Scroll (Pinch / Wheel)

Vertical scroll adjusts zoom:

zoom ← zoom × 0.9 (scroll down / pinch in) zoom ← zoom × 1.1 (scroll up / pinch out)

panY is scaled with zoom to maintain consistent feel: panY ← panY × (zoom_new / zoom_old).

onScaleUpdate (Two-finger drag)

Damped Jacobian for rotation and panY. Let delta = (dx, dy) be the gesture delta, and fDx, fDy be the finger separation components:

denom = fDy² + 30² rotation ← rotation − delta.dx · fDy / denom panY ← panY + delta.dy + delta.dx · fDx · fDy / denom

The 30² term prevents division by zero and dampens sensitivity when fingers are close together. Horizontal drag primarily rotates; vertical drag pans, with a cross-term from finger geometry.

onTapUp

Single tap: toggles UI visibility (e.g., hide/show top bar) or invokes _handleTap for object selection. Logic depends on tap target (sky vs. UI overlay).

Coordinate Transforms

Catalog (J2000) → JNow

Star and DSO catalogs use J2000.0 equatorial coordinates. Before projection, apply:

(RA_JNow, Dec_JNow) = equatorialEquinoxToJNow(RA_J2000, Dec_J2000, JD)

This applies precession and optionally nutation/aberration. JD is the current Julian Date.

Mount Positions

Mount RA/Dec (from :GR#, :GD# or bulk packets) are already in JNow. Use directly for crosshair and target marker projection.

Helper Functions

  • _parseLST — Parses LST string (e.g., from :GL#) to hours (0–24).
  • _parseCoord — Parses RA/Dec strings (e.g., :GR#, :GD#) to numeric radians or hours/degrees.

These handle LX200 format (e.g., +12*34:56 for Dec, 08:45:32 for RA).
