MainUnit Firmware

The TeenAstro MainUnit firmware runs on Teensy 3/4 microcontrollers and provides real-time mount control: motor drive, goto, sidereal tracking, multi-star alignment, encoder support, and LX200 serial protocol handling. It is the central controller for the telescope mount.

Source: TeenAstroMainUnit/

Application Class

The main application orchestrates initialization and the real-time control loop through setup() and loop().

setup() Sequence

Initialization runs in a fixed order to ensure hardware and state are ready before the main loop begins:

StepFunctionPurpose
1setupStartupBlink()LED blink sequence to indicate power-on and boot progress
2setupEepromAndMount()Load EEPROM, validate auto-init key, restore mount config and alignment matrix
3setupTimeSync()Initialize RTC/date-time, sync sidereal clock with GNSS or manual input
4setupReticule()Configure reticule LED and illumination if present
5setupMountPinsAndSidereal()Configure axis pins, start Timer1 for sidereal tick, enable step timers
6setupParkHomeAndGuideRates()Set park/home positions, guide rates, and tracking rates
7setupGnssProbe()Probe for GNSS/GPS module on serial port
8setupFocuserProbe()Probe for focuser module on serial port
9setupCommandSerial()Initialize LX200 command serial port (57600 baud)

loop() Sequence

The main loop repeatedly executes these phases in order:

PhaseFunctionPurpose
1loopCommandsAndStatus()Process serial commands, send status updates, handle bulk packets (GXAS/GXCS)
2loopSt4AndGuiding()Read ST4 guide port, apply guide pulses to axes
3loopEncoderSync()Read encoders, auto-sync mount position if enabled
4loopSiderealAndSafety()100Hz sidereal tick: update tracking targets, goto state machine, safety checks (altitude limits, collision)
setup() flow: setupStartupBlink ↓ setupEepromAndMount ↓ setupTimeSync ↓ setupReticule ↓ setupMountPinsAndSidereal ↓ setupParkHomeAndGuideRates ↓ setupGnssProbe ↓ setupFocuserProbe ↓ setupCommandSerial loop() flow (repeated): loopCommandsAndStatus ↓ loopSt4AndGuiding ↓ loopEncoderSync ↓ loopSiderealAndSafety

Key Methods

  • updateDeltaTarget() — Recomputes deltaTarget = target - pos
  • atTarget() — Returns true when pos equals target
  • resetToSidereal() — Sets interval to sidereal tracking rate
  • setSidereal() — Configures sidereal tracking
  • breakDist() — Returns braking distance for current speed
  • setIntervalfromDist() — Sets step interval from distance (for goto)
  • setIntervalfromRate() — Sets step interval from rate
  • move() — Initiates move to target with acceleration

Math Formulas

speedfromDist(d) = sign(d) · sqrt(2 · acc · |d|)

Speed (steps/s) from distance for constant acceleration.

ratefromSpeed(V) = V · interval_Step_Sid / ClockSpeed

Tracking rate from speed.

interval2speed(interval) = ClockSpeed / interval

Step interval to speed (steps/s).

Sidereal Clock & Timer ISRs

Master Clock

The sidereal clock is driven by Timer1 with:

masterClockSpeed = 1,000,000 Hz siderealClockSpeed ≈ 997,269.5625 Hz

The sidereal ratio compensates for Earth’s rotation: one sidereal day ≈ 23h 56m 4s.

Timer1 ISR

Timer1 fires at sidereal rate. On each tick:

  • m_lst++ — Increment local sidereal time counter
  • Update tracking targets for both axes
  • Update guiding state

TIMER3 / TIMER4

Each axis has a dedicated step pulse timer (TIMER3 for Axis1, TIMER4 for Axis2):

  • ISR fires at interval_Step_Cur (or sidereal interval when tracking)
  • Generates step pulse to motor driver
  • Updates pos and recomputes interval for acceleration/deceleration

Main Loop Sidereal Tick

The main loop runs a 100 Hz sidereal tick. On each onSiderealTick:

  1. Update tracking: target += fstep * elapsed for both axes
  2. Run goto state machine: check atTarget(), update intervals
  3. Safety checks: altitude limits, meridian flip, collision avoidance
  4. Update status for bulk packet (:GXAS#)
Timer hierarchy: Timer1 (sidereal) → m_lst, tracking target update TIMER3 (Axis1) → step pulse, pos update TIMER4 (Axis2) → step pulse, pos update Main loop (100 Hz): onSiderealTick → tracking, goto, safety

Tracking & Goto

Tracking

During sidereal tracking, the target advances at the sidereal rate:

target += fstep · elapsed

For both axes. fstep is the fractional step per tick; elapsed is the time since last tick.

Goto

Goto sequence:

  1. start = pos — Record current position
  2. target = axisTarget — Set target from sky coordinates (via alignment)
  3. moveTo() — Computes intervals from distance using acceleration profile

Acceleration

Acceleration is computed from maximum speed and distance for acceleration phase:

acc = Vmax² / (4 · degreesForAccel · stepsPerDegree)

This ensures smooth ramp-up and ramp-down within the acceleration distance.

Break Distance

Distance required to decelerate from current speed to zero:

breakDist = interval2speed(interval)² / (2 · acc)

Used to determine when to begin deceleration during goto.

The goto state machine uses trapezoidal velocity profiles: accelerate to max speed, coast, then decelerate to target.

Command Dispatcher

T_Serial Class

The command serial uses the LX200 frame format:

  • Format: :cmd# — colon, command string, hash terminator
  • ACK-only: \x06 (ASCII 6) for simple success
  • Some commands return data; length is defined in CommandReplyLength.h

processCommands

EEPROM Layout

Global Keys

  • EE_autoInitKey — Magic value to detect first run vs. valid config
  • EE_currentMount — Index of active mount profile
  • EE_sites — Site list (latitude, longitude, timezone)
  • EE_Mounts — Mount profiles (200 bytes each)

Per-Mount Block (200 bytes)

Each mount profile stores:

FieldDescription

alignment T matrix9 floats for CoordConv T
EEPROM writes are wear-limited. Config changes are batched and written only when necessary.

  • setRef() — Set reference position
  • calibrate() — Calibrate encoder scale

autoSyncWithEncoder

When enabled, the mount syncs stepper position to encoder when the error exceeds a tolerance:

ConstantTolerance
ES_601°
ES_300.5°
ES_150.25°
ES_70.125°

Sync occurs in loopEncoderSync() when |pos_steps - pos_encoder| exceeds the selected tolerance.
