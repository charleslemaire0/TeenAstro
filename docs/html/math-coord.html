Class Hierarchy

TeenAstroCoord defines four coordinate-system classes, all built on a common rotation-chain base. Each coordinate is internally represented as an ordered sequence of Euler rotations that, when composed, produce a 3×3 rotation matrix mapping the coordinate frame to a Cartesian reference.

LA3 (static linear algebra) ↑ Coord3R (3 SingleRotations: m_Eulers[3]) ├── Coord_EQ (FrE, Dec, −Ha) ├── Coord_HO (FrH, Alt, −Az+π) └── Coord_IN (Axis3, Axis2, −Axis1) Coord3R (same base) └── Coord_LO (Axis3, Axis2, +Axis1) ← note: positive Axis1

Coord3R — Base Class

class Coord3R : protected LA3 {
protected:
    SingleRotation m_Eulers[3];
public:
    Coord3R(SingleRotation R0, SingleRotation R1, SingleRotation R2);
    SingleRotation getE0();  // first rotation
    SingleRotation getE1();  // second rotation
    SingleRotation getE2();  // third rotation
};

The three rotations are always applied as R = Rx(E0) · Ry(E1) · Rz(E2), producing a 3×3 rotation matrix that defines the coordinate frame orientation.

Coord_EQ — Equatorial Coordinates

class Coord_EQ : protected Coord3R {
public:
    Coord_EQ(double FrE, double Dec, double Ha);
    double FrE();          // field rotation (equatorial)
    double Dec();          // declination (radians)
    double Ha();           // hour angle (radians, normalized)
    double direct_Ha();    // raw hour angle before normalization
    double Ra(double LST); // right ascension = LST − Ha

    Coord_HO To_Coord_HO(double Lat, RefrOpt Opt);
    Coord_IN To_Coord_IN(double Lat, RefrOpt Opt, const double(&misalignment)[3][3]);
    Coord_LO To_Coord_LO(const double(&trafo)[3][3]);
};

Euler Sequence

R_EQ = R᮵(FrE) · Rᵦ(Dec) · Rᵧ(−Ha)
  • FrE — Field rotation in the equatorial frame (usually 0 for standard mounts)
  • Dec — Declination: angle from celestial equator (−π/2 to +π/2)
  • Ha — Hour angle: angular distance west of the meridian (Ha = LST − RA)
Right ascension is derived: RA = LST − Ha, normalized to [0, 2π).

Coord_HO — Horizontal Coordinates

class Coord_HO : protected Coord3R {
public:
    Coord_HO(double FrH, double Alt, double Az, bool IsApparent);
    double FrH();           // field rotation (horizontal)
    double Alt();           // altitude (radians)
    double Az();            // azimuth from south (radians, normalized)
    double direct_Az_S();   // raw azimuth

    Coord_HO ToApparent(RefrOpt opt);    // apply refraction
    Coord_HO ToTopocentric(RefrOpt opt); // remove refraction
    Coord_EQ To_Coord_EQ(double Lat);
    Coord_IN To_Coord_IN(const double(&misalignment)[3][3]);
private:
    bool mIsApparent;  // tracks whether refraction has been applied
};

Euler Sequence

R_HO = R᮵(FrH) · Rᵦ(Alt) · Rᵧ(−Az + π)
  • FrH — Field rotation in the horizontal frame
  • Alt — Altitude: angle above the horizon (0 = horizon, π/2 = zenith)
  • Az — Azimuth: measured from South toward West (astronomical convention)
  • mIsApparent — Flag tracking whether this is apparent (refracted) or topocentric (geometric)

Coord_IN — Instrument Coordinates

class Coord_IN : protected Coord3R {
public:
    Coord_IN(double Axis3, double Axis2, double Axis1);
    double Axis3();        // third axis (field rotation)
    double Axis2();        // second axis (Dec-like)
    double Axis1();        // first axis (HA-like, normalized)
    double Axis1_direct(); // raw first axis

    Coord_HO To_Coord_HO(const double(&misalignmentinv)[3][3], RefrOpt Opt);
    Coord_EQ To_Coord_EQ(const double(&misalignmentinv)[3][3], RefrOpt Opt, double Lat);
};

Euler Sequence

R_IN = R᮵(Axis3) · Rᵦ(Axis2) · Rᵧ(−Axis1)

Instrument coordinates represent the physical mount axes. The misalignment matrix (from the alignment model) maps between HO and IN frames. For a perfectly aligned mount, this matrix is the identity.

Coord_LO — Local / Logical Coordinates

class Coord_LO : protected Coord3R {
public:
    Coord_LO(double Axis3, double Axis2, double Axis1);
    double Axis3();
    double Axis2();
    double Axis1();  // note: NOT negated

    Coord_EQ To_Coord_EQ(const double(&trafoinv)[3][3]);
};

Euler Sequence

R_LO = R᮵(Axis3) · Rᵦ(Axis2) · Rᵧ(+Axis1)

Note the positive sign on Axis1 (unlike EQ/HO/IN which negate it). LO coordinates are used for the logical mount frame, handling pier-side and the alignment transformation T/Tinv.

Conversion Formulas

EQ → HO (Equatorial to Horizontal)

R_combined = R_EQ · Rᵦ(π/2 − Lat) where R_EQ = R᮵(FrE) · Rᵦ(Dec) · Rᵧ(−Ha) Extract angles via getEulerRxRyRz(R_combined): FrH, Alt, Az_S → Az = π − Az_S

If refraction is enabled, Topocentric2Apparent is applied to Alt before constructing the Coord_HO result.

HO → EQ (Horizontal to Equatorial)

R_combined = R_HO · Rᵦ(−(π/2 − Lat)) where R_HO = R᮵(FrH) · Rᵦ(Alt) · Rᵧ(−Az + π) Extract via getEulerRxRyRz: FrE, Dec, raw_Ha → Ha = −raw_Ha

HO → IN (Horizontal to Instrument)

R_IN_matrix = R_HO_matrix · T where T is the 3×3 alignment matrix (HO → IN) Extract Axis3, Axis2, Axis1 via getEulerRxRyRz.

IN → HO (Instrument to Horizontal)

R_HO_matrix = R_IN_matrix · T¹ where T¹ = T⁻¹ (inverse alignment matrix) Extract FrH, Alt, Az via getEulerRxRyRz. If refraction used: Apparent2Topocentric on Alt.

EQ → LO (Equatorial to Logical)

R_LO_matrix = R_EQ_matrix · T⁻¹ Extract Axis3, Axis2, Axis1 via getEulerRxRyRz.

LO → EQ (Logical to Equatorial)

R_EQ_matrix = R_LO_matrix · T_inv Extract FrE, Dec, Ha via getEulerRxRyRz.
All conversions follow the same pattern: compose the coordinate’s Euler rotation chain, multiply by the latitude rotation or alignment matrix, then decompose back via getEulerRxRyRz.
