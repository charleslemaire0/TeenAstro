Flutter App

The TeenAstro Flutter app provides a cross-platform (Android, Windows) user interface for telescope control: connection, dashboard, goto, catalog browser, planetarium, alignment wizard, tracking, and settings. It connects to the mount via TCP (WiFi Server on port 9999) using the LX200 protocol.

Source: teenastro_app/

Riverpod State Management

State is managed with Riverpod:

  • MountStateNotifier — Holds MountState, polls GXAS, updates providers
  • mountStateProvider — Exposes current mount state to widgets

Other providers: connection state, catalog filter, planetarium view, night mode.

TCP Client (LX200TcpClient)

LX200TcpClient connects to the WiFi Server and sends LX200 commands:

MethodPurpose
connect(host, port)Open TCP socket to mount
disconnect()Close socket, cancel subscriptions
sendCommand(cmd)Send :CMD#, wait for reply
heartbeatFires every 1.5 s; sends :GXAS# if idle >1 s
mutexSerializes command send/receive to avoid interleaving
Disconnect order: close socket first, then cancel stream subscription, to avoid race conditions.

MountState and MountConfig

MountState (from GXAS parsing)

Fields include: tracking, sidereal, parkState, atHome, speed, pulseGuiding, guidingEW, guidingNS, trackCorrected, aligned, alignmentRefCount, alignPhase, alignStarNum, mountType, pierSide, gnssFlags, error, enableFlags, ra, dec, ha, az, alt, targetRa, targetDec, utcTime, utcDate, localTime, siderealTime, timezoneOffset, latitude, longitude, focuserStatus, focuserPos, trackRateRa, trackRateDec, storedTrackRateRa, storedTrackRateDec.

MountConfig (from GXCS parsing)

Per-axis motor config, rates, limits, encoders, refraction flags, mount index.

CatalogFilterState

CatalogFilterState holds filter selections for the catalog browser:

  • Constellation filter
  • Object type filter (galaxy, nebula, etc.)
  • Search text

Used by CatalogBrowserScreen to narrow the displayed catalog list.
