Linear Algebra Library (LA3)

TeenAstroLA3 provides all linear-algebra primitives used by the coordinate and alignment systems. It is a single static class (LA3) containing 28 public methods for 3-vector and 3×3 matrix operations, rotation matrices, Euler angle decomposition, atmospheric refraction, and singular value decomposition.

Authors: Markus Noga, Charles Lemaire
Source: libraries/TeenAstroLA3/

Types & Enumerations

RotAxis

enum RotAxis { ROTAXISX, ROTAXISY, ROTAXISZ };

Identifies which principal axis a rotation is performed around.

SingleRotation

struct SingleRotation {
    RotAxis axis;   // rotation axis
    double  angle;  // rotation angle (radians)
};

Pairs an axis with an angle to define one elemental rotation. Used as building blocks for multi-rotation chains (Euler angles).

RefrOpt

struct RefrOpt {
    bool   use;          // enable refraction correction
    double Temperature;  // ambient temperature (°C)
    double Pressure;     // atmospheric pressure (mbar)
};

Atmospheric conditions for refraction corrections. Passed to Topocentric2Apparent and Apparent2Topocentric.

Vector Operations

Dot Product

static double dotProduct(const double(&a)[3], const double(&b)[3]);
a · b = a₀b₀ + a₁b₁ + a₂b₂

Cross Product

static void crossProduct(double(&out)[3], const double(&a)[3], const double(&b)[3]);
a × b = ( a₁b₂ − a₂b₁, a₂b₀ − a₀b₂, a₀b₁ − a₁b₀ )

Norm (Euclidean)

static double norm(const double(&in)[3]);
‖v‖ = √( v₀² + v₁² + v₂² )

Normalize

static void normalize(double(&out)[3], const double(&in)[3]);
v̂ = v / ‖v‖

Angle Between Two Vectors

static double angle2Vectors(const double(&a)[3], const double(&b)[3]);
θ = acos( (a · b) / (‖a‖ · ‖b‖) )

Direction Cosines from Polar Angles

static void toDirCos(double(&dc)[3], double ang1, double ang2);

Converts two polar angles (azimuth θ₁, elevation θ₂) to a unit direction vector:

dc₀ = cos(θ₂) · cos(−θ₁) dc₁ = cos(θ₂) · sin(−θ₁) dc₂ = sin(θ₂)

This maps celestial coordinates (HA/Az, Dec/Alt) to Cartesian direction cosines on the unit sphere.

Rotation Matrices

static void getSingleRotationMatrix(double(&out)[3][3], const SingleRotation sr);
static void getMultipleRotationMatrix(double(&out)[3][3], const SingleRotation* sr, int n);

All rotations follow the right-hand rule: positive angle = counterclockwise when looking along the axis toward the origin.

Rx(θ) — Rotation about X-axis

[ 1 0 0 ] R᮵(θ) = [ 0 cos(θ) sin(θ) ] [ 0 −sin(θ) cos(θ) ]

Rotates in the YZ plane. Used for field rotation (FrE, FrH, Axis3).

Ry(θ) — Rotation about Y-axis

[ cos(θ) 0 −sin(θ) ] Rᵦ(θ) = [ 0 1 0 ] [ sin(θ) 0 cos(θ) ]

Rotates in the XZ plane. Used for declination/altitude (Dec, Alt, Axis2).

Rz(θ) — Rotation about Z-axis

[ cos(θ) −sin(θ) 0 ] Rᵧ(θ) = [ sin(θ) cos(θ) 0 ] [ 0 0 1 ]

Rotates in the XY plane. Used for hour angle/azimuth (HA, Az, Axis1).

Multiple Rotations

getMultipleRotationMatrix computes the product of n single-axis rotations applied left-to-right:

R = R₁ · R₂ · … · Rₙ

This is the foundation for all coordinate-system transformations in TeenAstro.

Euler Angle Extraction

Four decomposition routines extract Euler angles from a 3×3 rotation matrix. All handle gimbal lock (degenerate cases where |sin(θ)| = 1).

RxRyRz (Intrinsic X–Y–Z)

static void getEulerRxRyRz(const double(&r)[3][3],
    double& thetaX, double& thetaY, double& thetaZ);

Decomposes R = Rx(θX) · Ry(θY) · Rz(θZ). Primary decomposition used for coordinate conversions.

Given −1 < r₀₂ < 1: θY = asin(r₀₂) θX = atan2(−r₁₂, r₂₂) θZ = atan2(−r₀₁, r₀₀) Gimbal lock r₀₂ = −1: θY = −π/2, θX = 0, θZ = atan2(r₁₀, r₁₁) Gimbal lock r₀₂ = +1: θY = +π/2, θX = 0, θZ = atan2(r₁₀, r₁₁)

RzRyRx (Intrinsic Z–Y–X)

static void getEulerRzRyRx(const double(&r)[3][3],
    double& thetaZ, double& thetaY, double& thetaX);
θY = asin(−r₂₀) θZ = atan2(r₁₀, r₀₀) θX = atan2(r₂₁, r₂₂)

RzRxRy (Intrinsic Z–X–Y)

static void getEulerRzRxRy(const double(&r)[3][3],
    double& thetaZ, double& thetaX, double& thetaY);
θX = asin(r₂₁) θZ = atan2(−r₀₁, r₁₁) θY = atan2(−r₂₀, r₂₂)

Rx0RyRx1 (Proper Euler X–Y–X)

static void getEulerRx0RyRx1(const double(&r)[3][3],
    double& thetaX0, double& thetaY, double& thetaX1);
θY = acos(r₀₀) θX0 = atan2(r₁₀, −r₂₀) θX1 = atan2(r₀₁, r₀₂)

Atmospheric Refraction

Two complementary models from Meeus Astronomical Algorithms correct for atmospheric bending of light near the horizon.

Topocentric → Apparent (Saemundsson, Meeus Eq. 16.4)

static void Topocentric2Apparent(double& Alt, RefrOpt Opt);

Adds refraction to the true (geometric) altitude to get the apparent altitude:

Temperature-Pressure Correction: TPC = (P / 1010) · (283 / (273 + T)) Refraction (arcminutes): R = ( 1.02 / tan(h° + 10.3/(h° + 5.11)) + 0.0019279 ) · TPC Result: Alt_apparent = Alt_true + R · π / 10800 where h° = Alt in degrees, P = pressure (mbar), T = temperature (°C)
Not applied if Alt < −0.06 rad (≈ −3.5°) or Opt.use is false.

Apparent → Topocentric (Bennett, Meeus Eq. 16.3)

static void Apparent2Topocentric(double& Alt, RefrOpt Opt);

Subtracts refraction from the apparent altitude to recover the true altitude:

R = ( 1.0 / tan(h₀° + 7.31/(h₀° + 4.4)) + 0.0013515 ) · TPC Alt_true = Alt_apparent − R · π / 10800

Same TPC formula as above. Same guard conditions apply.

SVD Decomposition

static void getsvd(const double(&m)[3][3], double(&u)[3][3], double(&v)[3][3]);

Computes the Singular Value Decomposition of a 3×3 matrix. Implementation uses the McAdams et al. algorithm (UW-Madison Technical Report 1690, May 2011), optimized for 3×3 matrices with minimal branching.

Algorithm

  1. Normal equations: Compute ATA (symmetric 3×3).
  2. Jacobi eigenvalue iteration: Diagonalize ATA using approximate Givens rotations via quaternions. Constants: γ = √(8) + 3 ≈ 5.828, c* = cos(π/8), s* = sin(π/8). 4 sweeps over all three off-diagonal pairs (0,1), (1,2), (0,2).
  3. V matrix: Accumulate eigenvector quaternion → rotation matrix.
  4. B = A · V: Apply V to original matrix.
  5. Sort singular values: Sort columns of B by ρj = Σ(bij²) descending; apply same permutation to V.
  6. QR decomposition: Factor B = QR using Givens rotations (three steps).
  7. Output: U = Q, V = sorted right singular vectors. (Singular values S are not exposed.)
The SVD is used by CoordConv to project the raw Taki transformation matrix onto the nearest proper rotation (det = +1). See Alignment → SVD Correction.
